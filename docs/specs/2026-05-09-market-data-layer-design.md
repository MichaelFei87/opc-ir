# Market Data Layer Design

> **Status**: Approved
> **Date**: 2026-05-09
> **Scope**: Add real-time/historical asset prices and earnings data to OPC-IR's world model

---

## 1. Problem Statement

OPC-IR's world model currently relies entirely on RSS news events. It lacks:

1. **Macro asset prices and trends** — indices, yields, currencies, commodities, crypto
2. **Watcher asset detail** — individual stock prices, 52-week ranges, 3-month trends, quarterly earnings

These data are essential for watchers and strategists to produce grounded analysis.

## 2. Data Classification

### 2.1 Macro Instruments

| Category | Assets | Data Content |
|----------|--------|--------------|
| Equity indices | NDX, SPX, RUT, VIX, HSI, HSCEI, CSI300 | Latest price + 60-day trend |
| Currencies | DXY, CNH, USDCNY, CYB | Same |
| Commodities | GLD, WTI | Same |
| Crypto | BTC | Same |
| Yield curve | US3M, US2Y, US5Y, US10Y, US30Y | Latest yield + 60-day trend |
| Derived | 2s10s spread | Calculated as US2Y − US10Y |
| Policy rate | FFR | Maintained by econ-finance-watcher from Fed news (not API-fetched) |

### 2.2 Watcher Assets (equity-single)

Current list from `watch-assets.yaml`: MSFT, NVDA, GOOGL, META, TSM.

| Data | Frequency |
|------|-----------|
| Current price | Every evolve cycle |
| 52-week high/low, 3-month trend direction | Every evolve cycle |
| Latest quarterly earnings (EPS, revenue, guidance, YoY) | On earnings release |
| LLM earnings interpretation | Generated by corp-fundamentals-watcher during evolve |

## 5. New Scripts

### 5.1 bin/fetch-market-data.sh

- Embedded Python using `import yfinance`
- Batch-fetches all macro instruments + watcher assets in one call
- Computes derived metrics: 2s10s spread, trend directions, 60-day range
- Trend logic: compare current vs 7d-ago and 30d-ago to classify as up/down/sideways
- Writes `macro-snapshot.json` and `watcher-snapshot.json`
- Archives daily snapshot to `history/`
- Dependency: `pip3 install yfinance` (checked at runtime with install prompt)
- Fallback: if yfinance fails for a ticker, falls back to Yahoo raw API (urllib)

### 5.2 bin/fetch-earnings.sh

- Embedded Python using yfinance `.quarterly_financials` and `.earnings_dates`
- Only runs for `class: equity-single` assets from watch-assets.yaml
- Writes per-quarter JSON to `earnings/{SYMBOL}-{YYYY}Q{N}.json`
- LLM interpretation is NOT done in this script — handled by corp-fundamentals-watcher during evolve
- Trigger: called by evolve chain; skips if latest quarter data already exists

## 6. Pipeline Integration

### 6.1 Evolve Chain (Modified)

```
Current:  fetch-rss → triage → watchers → synthesize → world-model.md

New:      fetch-rss → triage → fetch-market-data → fetch-earnings → watchers → synthesize → world-model.md
                                     ↓                    ↓
                              macro-snapshot.json   earnings/*.json
                              watcher-snapshot.json
                                     ↓                    ↓
                              ─────── injected as watcher context ───────
```

### 6.2 Watcher Context Injection

- **econ-finance-watcher**: receives full `macro-snapshot.json` (yield curve, indices, currencies)
- **energy-commodity-watcher**: receives GLD, WTI, BTC sections from macro-snapshot
- **corp-fundamentals-watcher**: receives `watcher-snapshot.json` + latest `earnings/*.json`; generates LLM earnings summaries and writes them back to watcher-snapshot
- Other watchers: receive relevant macro-snapshot sections as read-only context

## 8. Error Handling

- If yfinance is not installed: script prints install instructions and exits 1
- If a ticker fetch fails: log warning, skip that asset, continue with others
- If all tickers fail: exit 1, evolve chain continues without market data (graceful degradation)
- Stale data: if snapshot is >24h old, watchers receive a staleness warning in their context
- Network timeout: 15s per batch request, 2 retries with exponential backoff

## 9. Testing

- Fixture mode: `OPC_IR_MARKET_FIXTURE_DIR` env var points to pre-built JSON fixtures
- Unit tests: validate JSON schema of snapshot outputs
- Integration test: fetch-market-data with fixture dir, verify watcher context injection
- Earnings test: verify skip-if-exists logic, validate earnings JSON schema

## 10. Scope Boundaries

**In scope:**
- fetch-market-data.sh and fetch-earnings.sh scripts
- Evolve chain integration (protocol update + watcher context injection)
- Market Dashboard injection into world-model.md
- watch-assets.yaml additions for yield curve
- Tests with fixtures

**Out of scope:**
- Premium data sources (Alpha Vantage, Polygon, etc.)
- Real-time streaming prices
- Technical analysis indicators (RSI, MACD, etc.)
- Portfolio tracking or P&L
